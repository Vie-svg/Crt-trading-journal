Règles CRT
Candle Range Theory — Framework complet RomeoTPT / TradesbyVee
Règles absolues — Non négociables
01Jamais d'entrée sans confirmation de close de bougie C3. Une bougie en mouvement n'est pas une confirmation. Attendre la clôture complète, sans exception.
02Setup sans Key Level = invalide. Un FVG, HOD/LOD, Weekly High/Low ou Equal Highs/Lows doit être présent et actif à proximité de l'entrée.
03Stop Loss au-delà du point Turtle Soup. Jamais dans le corps de C2. Jamais dans le wick. Le SL protège contre une invalidation réelle du setup.
04TP1 à 50% de C1 est obligatoire. Clôturer 50% de la position ici. Déplacer le SL au point mort immédiatement après. Cette règle ne se négocie pas.
05Risque maximum 0.5%–1% du capital par trade. Aucune exception, même pour un "setup parfait". La survie du compte passe avant le gain.
06R:R minimum 1:2 calculé avant d'ouvrir. Si le R:R n'est pas là, le trade n'existe pas. Forcer un trade avec R:R < 1:2 est une erreur de gestion.
07Checklist minimum 7/10 requis. En dessous = pas de trade, même si le setup paraît évident. La discipline est le seul edge qui ne disparaît pas.
08SMT = filtre de confirmation, pas signal d'entrée seul. La divergence SMT confirme le setup CRT, elle ne le crée pas. Sans C1/C2/C3 complet, le SMT ne suffit pas.
09KOD prioritaire sur C3 direct. Attendre le pullback est supérieur à chasser la continuation. Meilleur R:R, meilleure probabilité, moins de stress. Si KOD disponible, ne pas ignorer.
103 erreurs fatales codifiées : Surrisque (> 1%), revenge trading après une perte, ignorer la checklist. L'une d'elles active = pause immédiate de la session.

H1Confirmation. C3 en cours confirmé, Key Level actif validé, comportement du prix autour du niveau clé.
M15Timing d'entrée. Close de bougie C3, killzone active, séquence 1-5-9 H4, positionnement précis du SL.
Modèles d'entrée CRT — Hiérarchie
KODKiss of Death — Modèle prioritaire. Attendre le pullback vers la zone C3 après la close. Placer un ordre limite au niveau de retour. Plus haute probabilité, meilleur R:R, moins de stress d'exécution. Toujours privilégié quand disponible.
#1Model #1 — C3 close directe. Entrée sur la close de C3 sans attendre le pullback. Exige une confluence maximale (score ≥ 8/10). R:R potentiellement plus élevé mais risque de pullback vers l'entrée important.
C3C3 Direct sans pullback — Agressif. Réservé aux configurations HTF majeures uniquement. Risque réduit à 0.5% obligatoire. Ne jamais moyenner si le prix revient contre la position.

Guide d'utilisation
Comment déployer ce journal sur GitHub Pages
Données sauvegardées localement
Tous tes trades sont sauvegardés dans le localStorage de ton navigateur. Ils persistent entre les sessions. Exporte régulièrement en JSON comme backup.
Déploiement GitHub Pages — Étapes complètes
01Créer un compte GitHub. Va sur github.com → Sign up → Entrer email + mot de passe + nom d'utilisateur → Confirmer l'email. Gratuit.
02Créer un nouveau repository. Clic sur le "+" en haut à droite → New repository → Nom : crt-journal → Visibility : Public → Cocher "Add README" → Create repository.
03Uploader les fichiers. Dans le repo → Add file → Upload files → Glisser tout le dossier du journal (index.html + dossiers css/ et js/) → Commit changes.
04Activer GitHub Pages. Settings → Pages (menu gauche) → Source : Deploy from a branch → Branch : main → / (root) → Save.
05Obtenir l'URL. Attendre 2–3 minutes → Revenir sur Settings → Pages → L'URL apparaît : https://tonnom.github.io/crt-journal
06Ajouter en favori / raccourci. PC : Ctrl+D pour sauvegarder le favori. Téléphone : Ouvrir l'URL dans Chrome/Safari → Menu partage → "Ajouter à l'écran d'accueil". Le journal s'ouvre comme une app.
07Mise à jour. Pour mettre à jour après modification : Aller sur github.com → ton repo → cliquer sur le fichier → icône crayon → modifier → Commit changes. GitHub Pages se met à jour en 1–2 minutes.
Backup et récupération des données
B1Export CSV → Bouton "Export CSV" sur le Dashboard. Ouvre dans Excel ou Google Sheets pour analyser. Format : UTF-8 avec BOM (compatible Excel français).
B2Backup JSON → Bouton "Backup JSON". Sauvegarde tous les trades en JSON. À faire chaque semaine. Garder sur Google Drive ou Notion.
B3Import JSON → Bouton "Importer JSON" sur le Dashboard. Pour restaurer sur un nouvel appareil ou après effacement. Les doublons sont automatiquement ignorés.
⚠Important : Les données sont dans le localStorage de ton navigateur. Vider le cache du navigateur efface les données. Toujours faire un backup JSON avant de vider le cache ou changer d'appareil.
